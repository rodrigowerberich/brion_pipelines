Using cmake you can build following these instructions\+: Assuming you are running from this projects root. 
\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/B\ build}
\DoxyCodeLine{cmake\ -\/-\/build\ build\ -\/j\ 14}

\end{DoxyCode}


The compiled binary will be {\ttfamily brion\+\_\+pipelines/build/components/pipeline\+\_\+parser} But it can be installed on your system by calling cmake install 
\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/-\/install\ build\ -\/-\/prefix\ ./}

\end{DoxyCode}
 This will create a folder called bin in your current directory and the executable pipeline\+\_\+parser will be inside it.

For the -\/j option (parallel building), use whatever is more efficient on your machine.

You will be able to find the more complete documentation in the docs folder. Please use the html version, the latex was not revised.

To open the documentation you can open docs/html/index.\+html.\hypertarget{Readme_autotoc_md21}{}\doxysection{\texorpdfstring{How to run?}{How to run?}}\label{Readme_autotoc_md21}

\begin{DoxyCode}{0}
\DoxyCodeLine{bin/pipeline\_parser\ <file\_name>}

\end{DoxyCode}


A small help with some further options can be seen by passing -\/h 
\begin{DoxyCode}{0}
\DoxyCodeLine{bin/pipeline\_parser\ -\/h}

\end{DoxyCode}

\hypertarget{Readme_autotoc_md24}{}\doxysection{\texorpdfstring{Running over docker}{Running over docker}}\label{Readme_autotoc_md24}
If you want to build using docker you can run, with this repo as the working directory


\begin{DoxyCode}{0}
\DoxyCodeLine{docker\ build\ .\ -\/t\ brion\_pipeline:latest}

\end{DoxyCode}


If it worked, you should see the help message if you run 
\begin{DoxyCode}{0}
\DoxyCodeLine{docker\ run\ brion\_pipeline:latest}

\end{DoxyCode}


To parse a file, e.\+g example 1, execute\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{docker\ run\ -\/v\ ./examples/test\_file1.txt:test\_file1.txt\ brion\_pipeline:latest\ ./bin/pipeline\_parser\ test\_file1.txt}

\end{DoxyCode}
